aiGradeMe is a compact Python web application that uses AI to grade photos of hand-drawn floor plans based on requirements defined in a rubric. The entire project lives in just four files and runs reliably on Renderâ€™s free tier.

The backend is written in pure Python with Flask (appnew.py). It exposes three simple routes: a keep-alive endpoint, a /rubric endpoint that presents the grading rules to the user, and the main /submit POST route. When a user uploads their sketch, Flask validates the form data, securely saves the image to a temporary file, and immediately passes the path to a dedicated helper module.

That helper, aichecknew.py, handles all interaction with the Gemini API. The helper program reads the image from disk, base64-encodes it, and constructs a multimodal request containing both the full grading prompt (loaded from prompt.txt) and the encoded image. The request is then sent to Googleâ€™s generativelanguage.googleapis.com endpoint for the gemini-2.5-flash model. The code includes retry logic with exponential back-off (up to four attempts) to gracefully handle temporary rate limits or network glitches. On success, Gemini returns a JSON response containing six rubric scores in â€œx/yâ€ format plus a short written feedback string.

The prompt itself (stored in prompt.txt) is split into two sections. The top section has the public rubric shown to the user. Everything after the rubric is the precise system prompt Gemini receives, which explicitly defines the six grading categories, their point values, scoring guidelines, and the exact JSON structure expected in the reply. This careful prompt engineering ensures consistent, parseable output even when the model occasionally varies label names.

Back in the main application (appnew.py), JSON is extracted from Geminiâ€™s response, the six score entries are taken regardless of key order, and each â€œx/yâ€ value is converted to an integer. The backend then sums the points to produce a total out of 100 and dynamically generates a complete HTML grade report containing the userâ€™s name, the large total score, a table of all six categories with their exact labels and points, and the AIâ€™s feedback. This HTML is returned directly to the browser and injected into the page via JavaScript, replacing the upload form instantly.

The frontend (aigrademe.html) is a single, responsive static page with vanilla JavaScript that posts the form data straight to the Flask endpoint and handles the loading and result states. Images are deleted immediately after processing, and any error at any step returns a clear, friendly message instead of a crash.

One thought for a future upgrade is to modify aichecknew.py to fallback to another vision model when Gemini fails (by adding a second API call with its own key), as Gemini does get overloaded frequently. Another upgrade would be to retain the user info and AI results to a database for future review. The external elements of the application (frontend, prompt handling, result display) would not need modification.

While created to review hand-drawn floor plans, this program is powerful enough to be used for virtually anything. I designed the project from the ground up to be a reusable AI grading template. To turn it into a grader for any other visual assignment (such as circuit diagrams, chemistry mechanisms, geometry proofs, fashion sketches, or anything else users can photograph), we only need to edit one file: prompt.txt. Replace the top half with the new public rubric and the bottom half (after === PROMPT ===) with the new instructions for Gemini, keeping the same six-category, 100-point structure and the required JSON output format. The Flask backend, AI integration, image handling, and result display already accept any category names and scores the model returns, and the HTML page is deliberately generic (â€œupload your work,â€ â€œgrade my sketchâ€), so no other code changes are required.
